## Google Analytics

1. Copy `.env.example` to `.env.local`.
2. Set your GA4 Measurement ID for client-side tracking:

```bash
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
```

3. Set the credentials for the admin analytics dashboard (Google Analytics Data API):

```bash
GA4_PROPERTY_ID=123456789
GOOGLE_CLIENT_EMAIL=your-service-account@your-project.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nYOUR_PRIVATE_KEY_HERE\n-----END PRIVATE KEY-----\n"
```

4. Restart the dev server.

When `NEXT_PUBLIC_GA_ID` is set, Google Analytics is automatically injected in the root layout.

The admin dashboard reads real analytics from the Google Analytics Data API. It requires the GA4 property id and a service account with access to that property. If those variables are missing, the analytics route returns an error instead of fallback data.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.
